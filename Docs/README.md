# Docs README

This folder contains living documentation for the Study Planner project (Flutter + Firebase).
All documents here must be updated after each session.

## Contents
- `Progress_Report.md`: Session-by-session progress, tests, and errors.
- `API_Routes.md`: Cloud Functions + Firestore data contracts.
- `Frontend_Pages.md`: UI pages, routes, and key components.
- `Sessions/`: Detailed session logs with atomic tasks and tests.

## Update rules
- After each work session, update `Progress_Report.md` and add a new session file in `Sessions/`.
- If functions/rules/schema change, update `API_Routes.md` the same session.
- If UI pages/routes/components change, update `Frontend_Pages.md` the same session.
